# 用 TODO + 管家做工作总结

如果你希望稳定地产出日报、周报，最实用的方法是：

1. 在首页 TODO 持续记录和勾选任务
2. 用管家（Butler）把任务沉淀为结构化“工作台账”
3. 让管家按时间段自动整理为日报/周报

## 适合什么场景

- 每天任务很多，收尾时想快速出日报
- 每周需要固定提交周报
- 想把“做了什么、结果是什么、下一步是什么”沉淀成长期记录

## 一次性准备（建议 5 分钟）

## 1. 先把首页 TODO 用起来

建议每条 TODO 都按这个格式写：

- 动作 + 对象 + 结果
- 示例：`完成支付链路联调并输出联调结论`

这样后续总结更清楚，不会只有“开会、沟通、处理问题”这类空泛描述。

## 2. 在聊天中切换到“管家”

在输入框输入 `@`，选择 `管家（Butler）`，然后发送下面这段初始化指令：

```text
请帮我创建一张“工作台账”并长期维护。字段至少包含：
日期、任务、状态（完成/进行中/阻塞）、产出结果、关联人/项目、风险与阻塞、下一步。
后续我给你当天 TODO 和进展时，你都更新这张台账，不要重复建新表。
```

目的：让管家后续每次都往同一份台账里追加和更新。

## 每日使用流程（3 步）

## 1. 白天更新首页 TODO

- 新增任务
- 勾选完成项
- 给关键任务补一句结果（建议）

## 2. 下班前对管家说“同步今天工作”

把今天的关键进展告诉管家，让它更新台账：

```text
请把我今天（2026-03-04）的工作更新到工作台账：
1) 完成支付链路联调，输出结论文档
2) 修复登录态丢失问题，已上线
3) 性能优化做到一半，阻塞在压测环境权限
并按“完成/进行中/阻塞”标记状态。
```

## 3. 直接让管家生成日报

```text
请基于工作台账生成今天的日报（2026-03-04）：
1) 今日完成
2) 今日问题与风险
3) 明日计划
要求：简洁、可直接发给主管。
```

## 周报使用流程（建议每周五）

```text
请基于工作台账生成本周周报（2026-03-01 到 2026-03-07）：
1) 本周完成事项（按项目归类）
2) 关键成果与影响
3) 未完成项与原因
4) 下周计划
5) 需要协同/资源支持
输出为“可直接复制到飞书/企业微信”的格式。
```

## 月报/阶段复盘模板

```text
请基于工作台账生成 2026 年 3 月工作复盘：
1) 核心目标与达成情况
2) 代表性成果（3-5条）
3) 主要问题与改进动作
4) 下月重点计划
格式偏管理汇报风格。
```

## 提升总结质量的 5 个技巧

1. 每条 TODO 尽量写“结果”，不是只写“动作”  
2. 阻塞项一定写“原因 + 需要谁支持”  
3. 让管家固定输出结构（日报/周报模板保持一致）  
4. 每天都做一次“同步台账”，不要攒一周再补  
5. 提问时写清楚时间范围（例如 `2026-03-01 到 2026-03-07`）

## 常见问题

### 管家输出太空泛怎么办？

在指令里加限制：

- “每条成果都要有结果证据（文档、上线、指标）”
- “不要写泛化表述，优先写可验证结果”

### 周报里漏了任务怎么办？

让管家先做一步“查漏”再生成周报：

```text
先检查本周台账中是否有未归类或缺少结果的任务，补全后再生成周报。
```

### 我有多个项目怎么办？

让管家按“项目名”分组输出：

```text
请按项目A/项目B分组输出本周完成事项与风险。
```
