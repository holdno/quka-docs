- chunkservice 文档chunk服务

# 开始  
项目根目录下`docker-compose`文件夹中有定义好的 docker compose 配置文件  
可以复制 docker-compose.yml 配置内容到本机项目目录  
假设项目目录为 `/data`  
为了避免目录权限问题，需要用户预先在项目目录中创建数据库目录与文件存储目录
```shell
cd /data
mkdir postgres  
mkdir minio  
```
这两个目录的挂载信息可以在 docker-compose.yml 中查看  
目录创建好后即可通过命令 `sudo docker-compose -f docker-compose.dev.yml up -d` 来安装服务  
安装完成后，可以通过 docker logs qukaai 查看主服务运行状态，并搜索关键词 `Access token` 找到初始管理员的AccessToken信息(首次运行时由系统自动创建)，切记要保管好该 Token，作为后续系统登录的凭证  
  

# 🌟 通过白山智算平台，快速激活{process.env.SITE_NAME}的AI能力  
(免费!快薅!)

{process.env.SITE_NAME}已经内置了白山云的快捷配置模板，让你能够轻松接入白山云的AI服务。

配置步骤

1. 登录系统：访问QukaAI管理界面，选择Selfhost模式登录，并使用上述管理员AccessToken进行登录
2. 进入模型配置页面：
    - 点击左下角头像 → "AI 模型管理" → "模型提供商"
    - 点击 "添加新提供商" 按钮
3. 选择白山云模板：
    - 在弹窗中的 “快速开始模版” 选择 "白山云"，系统将自动加载预置的配置模板
        ![快速创建白山云接入](/deployment/model_provider_tpl.png)
4. 填写配置信息：
    - API密钥：填入你的白山云API密钥（获取API密钥：登录[白山智算平台](https://ai.baishan.com/auth/login?referralCode=Ruh0hYhjvM) > 个人中心 > API密钥）
5. 配置好模型供应商后，通过顶部tab进入到"模型配置"管理页
    - 通过添加模型配置下述模型
    - 供应商选则刚刚创建的 "白山云"，依次创建以下类型的模型配置(下述为推荐模型，具体可根据个人需求进行配置)  
        - 聊天类型：Qwen3-32B-FP8  
        - 思考聊天类型：GLM-4.5
        - 向量化类型：BAAI/bge-m3  
        - 重排序类型：BAAI/bge-reranker-v2-m3  
        - 增强类型：Qwen3-32B-FP8 
6. 配置好模型后，通过tab进入到"使用配置"管理页：
根据上述配置再次配置为当前使用的模型
