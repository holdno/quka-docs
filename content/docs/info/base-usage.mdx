在设计上，{process.env.SITE_NAME}的数据组织结构分为两级，空间 -> 资源类型。

## 数据结构

### 空间

- 不同空间的内容互相隔离，数据无法跨空间流动。
  - 用户可以将不同事业的记忆组织到不同的空间中，例如个人的记忆是一个空间，某个事业上的记忆是另一个空间。
- 每个空间可以有不同的用户加入。
  - 例如实现知识共享群组等。

用户在注册时，会创建第一个空间，注册完成后可以在用户首页继续添加新的空间。

#### 注册时需填写第一个空间名称

可以命名为 主空间 或 个人空间。

### 资源类型

给空间中不同的内容添加一个资源标记，类似于电脑中的文件夹。

资源类型在创建时，会有一个“清理周期(天)”设置，意思为从该记忆被创建时起，经过 N 天后自动删除该记忆，填 0 则系统不自动清理。
我们可以根据各人需求创建出 30 天 90 天 180 天 365 天等不同清理逻辑，来使系统帮我们自动清理一些无用的内容(指某些记忆可能过了某一天就不具备任何价值)。

![资源类型创建入口](/资源类型创建入口.png)

- ID：资源唯一标识，仅支持大小写，用于系统识别资源类型，标记记忆属于哪个资源。常见于 API 的使用场景中
- 标题：展示在 UI 界面中的名称
- 清理周期：表示该资源类型下的记忆，自创建起请过 N 天后自动被系统删除，避免过期记忆过多导致后续 AI 将面临噪音问题  
  0 或不填则表示永久记忆，不会被系统自动清理
- 描述：对资源的定义，仅提供用户个人参考
